# 收藏功能 UI/UX 设计

_决定于 2026-04-29，与 Jason 反复讨论后定稿。_

## 核心原则

**分层访问（freemium）**：
- 浏览所有内容 → 无任何门槛
- 收藏 / 我的收藏 → 需要飞书登录

**登录是飞书 OAuth 唯一入口**：
- 没有飞书账号的访客可以正常浏览，仅无法使用收藏功能
- 不强推注册，但提供出口

---

## 决策记录

### A. 弹窗触发时机：首次完整 + 后续简版 + 顶部入口

**首次未登录用户点 ⭐**：
- 弹完整引导对话框（解释为什么要登录、有什么好处）
- 用户关闭后用 `localStorage` 记住"已经看过完整版"

**之后未登录用户点 ⭐**：
- 只弹简版（一句话 + 登录按钮）
- 不再重复完整解释

**顶部 nav 始终有"登录"入口**：
- 用户随时能找到登录入口，不依赖点击 ⭐ 触发

### B. 文案语气：友好温和

**完整版弹窗文案**：
> ⭐ 收藏功能需要登录
>
> 登录后才能保存你喜欢的内容，跨设备同步。
>
> [使用飞书登录]
>
> 没有飞书账号？[去注册](https://www.feishu.cn/signup) | [继续浏览]

**简版提示**（toast 或迷你弹窗）：
> 登录后才能收藏。[使用飞书登录]  [×]

### C. "去注册飞书"入口：小字文字链接

- 不放醒目按钮
- 在弹窗底部一行小字链接，外链到 `https://www.feishu.cn/signup`
- 新窗口打开（target="_blank"）

### D. 登录后自动收藏：是

**实现要点**：
- 用户点击 ⭐ 时，把目标资讯的 `itemId` 存到 `sessionStorage`（key: `pendingFavorite`）
- 走完飞书 OAuth 跳回网站后，前端启动时检查 `pendingFavorite`：
  - 有 → 自动调用收藏 API → 删除 sessionStorage 标记 → 显示 toast"已收藏"
  - 无 → 正常进入页面

---

## 边界情况

### 飞书 OAuth 失败

- 用户在飞书页点"取消"
- 飞书账号被组织禁用
- 网络中断

**统一处理**：跳回网站后顶部显示轻量 toast：
> 登录未完成，你可以稍后再试

不弹大对话框打扰用户。

### 已登录用户的额外 UI 元素

- 顶部 nav：显示飞书头像 + 姓名 + "我的收藏"入口
- 已收藏的资讯：⭐ 按钮变成实心 ⭐
- 点已收藏的 ⭐ → 取消收藏（同一按钮二次切换）
- 设置页（小齿轮 / 头像下拉）：
  - 显示当前登录身份
  - "退出登录"按钮
